# Forge Network Branding Rules

This document defines what you may and may not do with the **Forge Network** name, logo, and credits when forking or modifying **OpenMark**.


## 1. Project Name

- **Allowed:**
  - Rename the project for your own use (e.g., `MyCustomClient`, `RPServer`, etc.).
  - Add a note such as: “Based on OpenMark (Forge Network).”

- **Not allowed:**
  - Present yourself as “Forge Network” or as an official Forge Network project.
  - Remove all traces of Forge Network as the original creator.


## 2. Logos and Icons

- **You may:**
  - Change the main icon of your fork.
  - Add your own branding alongside Forge Network credits.

- **You may not:**
  - Modify a Forge Network logo and claim it as your own.
  - Use Forge Network logos to imply official support.


## 3. Mandatory Credits

Your fork or derivative must:

1. Keep a mention in the `README`:

   > This project is a fork/derivative of **OpenMark**, created by **Forge Network** (https://forgenet.fr).

2. Keep the `NOTICE.md` file intact (you may append additional notes).

3. Never remove mandatory Forge Network credits in “About” screens, credit files, or documentation.


## 4. Common Scenarios

- **Servers, mods, or custom clients:**  
  You may rename and rebrand, but you **must** keep at least one visible credit to Forge Network.

- **Repack or redistribution:**  
  You must include `LICENSE.md`, `NOTICE.md`, and ideally `BRANDING.md`.

For advanced use of the Forge Network name or logo (marketing, partnerships, etc.), explicit permission is required.
